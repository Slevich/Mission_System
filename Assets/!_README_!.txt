Опишу, что было сделано в проекте, чтобы проще было ориентироваться!

Есть интерфейс IMission. Он носит в себе реализацию операций с миссией. Там методы запуска, окончания и т.д.
При этом запуск может быть как немедленный, так и с задержкой, если указать её в параметре метода.

Есть класс MissionData. Он содержит информацию о шаге + в нем содержится ссылка на реализацию.
При этом в качестве реализации подойдет либой MonoScript, реализующий интерфейс IMission. Он настраивается в инспекторе.
На основе класса из MonoScript создается экзепляр, который и записывается в IMission-поле у MissionData.

Есть MissionSequenceScriptable. Это по сути описание последовательности и миссий, которые в неё входят. Последовательность миссий идет сверху вниз.
Плюс там есть кнопки для разных манипуляций с элементам. Увы, кнопку вставить какой либо элемент между двумя другими не успел сделать.
Есть конкретный ScriptableObject - называется "TestSequence". Там и описаны все миссии.

Есть MissionManager. Он содержит в себе список последовательностей, с дополнительной информацией о них (класс MissionsSequence). Сам менеджер просто вызывает нужный метод у нужной последовательности (по индексу).
MissionSequence получает список шагов из MissionSequenceScriptable и управляет ими (запускает и завершает).
Здесь можно смотреть за текущим состоянием шагов.
Здесь лучше вообще ничего не трогать, кроме как управлять последовательностями. ВНУТРИ ПОСЛЕДОВАТЕЛЬНОСТИ ПОЛЯ НЕ МЕНЯТЬ, ИНАЧЕ МОЖЕТ СЛОМАТЬСЯ. Настраивать всё из MissionSequenceScriptable. 

Есть NotificationController. Это просто класс, проигрывающий анимацию для UI и получающий данные из MissionSequence об измении статуса текущей миссии.

Плюс там много кода в редакторе. Да он кривоватый и есть, что даделать, но функциональность несёт.

В задании был акцент, что у миссии должна быть указана связь с другими миссиями. У меня они идут просто друг за другом.
Такие ссылки имеют смысл, если используется, например, нодовая система, в которой все миссии и настраиваются. В стандартном редакторе Unity - это решение не лучшее, лишь создает визуальный шум (имхо).

Хотел еще сделать, чтобы выводилась последовательность с текущей миссией справа сверху, но, увы, не успел.
